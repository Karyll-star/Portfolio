# 背景高度扩展 - 覆盖下方元素

## 改进概述
增加背景高度，使其能为首屏、分割线以及部分下方元素（如作品集）提供视觉背景支撑。

## 具体修改

### 1. 桌面端（默认）
**修改前：**
```css
.bglight {
    height: 720px;  /* 仅覆盖首屏 */
}
```

**修改后：**
```css
.bglight {
    height: 1200px;  /* 覆盖首屏 + 下方作品集等元素 */
}
```

**说明：**
- 增加 480px（从 720px 到 1200px）
- 现在背景可以延伸到作品集部分
- 为整个首屏区域及部分内容提供统一的视觉背景

### 2. 手机端（< 767px）
**修改前：**
```css
@media (max-width: 767px) {
    .bglight {
        min-height: 400px;  /* 较低的最小高度 */
    }
}
```

**修改后：**
```css
@media (max-width: 767px) {
    .bglight {
        min-height: 600px;  /* 大幅增加 */
    }
}
```

**说明：**
- 增加 200px（从 400px 到 600px）
- 在手机上，背景现在能覆盖首屏标题、副标题、社交链接以及部分下方内容
- 保持视觉的连贯性

### 3. 极小屏幕（< 479px）
**修改前：**
```css
@media (max-width: 479px) {
    .bglight {
        min-height: 300px;  /* 较小的最小高度 */
    }
}
```

**修改后：**
```css
@media (max-width: 479px) {
    .bglight {
        min-height: 450px;  /* 增加 150px */
    }
}
```

**说明：**
- 增加 150px（从 300px 到 450px）
- 在极小屏幕上也能提供足够的背景覆盖
- 确保标题和导航都有背景支撑

## 技术细节

### 为什么增加背景高度？
1. **视觉连贯性** - 背景与内容形成统一的视觉块
2. **区域划分** - 清晰地分离首屏区域和下方内容
3. **设计层次** - 作品集等内容在背景衬托下更显眼

### 为什么不覆盖所有内容？
1. **设计考量** - 不同区域可能有不同的视觉设计
2. **内容分层** - 给后续内容保留独立的视觉空间
3. **性能考虑** - 过大的背景元素可能影响渲染性能

### 为什么各端口高度不同？
1. **桌面（1200px）** - 相对宽敞，背景可以承载更多内容
2. **手机（600px）** - 空间有限，但仍需覆盖重要内容
3. **极小（450px）** - 极端情况下的最小保障高度

## 使用 `height: auto + min-height` 的好处

```css
@media (max-width: 767px) {
    .bglight {
        height: auto;         /* 根据内容自动调整 */
        min-height: 600px;    /* 但最小不低于 600px */
    }
}
```

**优点：**
- ✅ 自适应性强，内容少时不会浪费空间
- ✅ 内容多时自动扩展，不会被截断
- ✅ 保证最小可见高度
- ✅ 流式布局更灵活
