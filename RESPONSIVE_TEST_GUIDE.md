# 响应式设计验证指南

## 🎯 验证目标
确保在所有屏幕宽度下（从 320px 到 1920px），网页：
- ✅ 无横向溢出和滚动
- ✅ 文字清晰可读
- ✅ 卡片正常堆叠
- ✅ 分界线正常显示（或隐藏）
- ✅ 布局流畅过渡
- ✅ 桌面端原样不变

---

## 📱 测试断点

### 1. **极小屏幕（< 480px）** - 来自 @media (max-width: 479px)
**代表设备**: iPhone SE (375px), 老旧手机

**应该看到**:
- [ ] 导航栏：logo + 下载按钮水平排列
- [ ] 首屏标题：32-36px 大字
- [ ] 首屏内容：竖排堆叠，100% 宽度
- [ ] 3D 模型：240px 高，100% 宽
- [ ] 作品卡片：竖排单列，每卡宽度 100%
- [ ] 社交链接：flex 排列，自动换行
- [ ] 无横向滚动条
- [ ] 边距：左右各 12px

**测试步骤**:
```
1. 打开浏览器 DevTools（F12）
2. 选择 Ctrl+Shift+M 进入响应式设计模式
3. 调整宽度为 375px
4. 或选择预设 "iPhone SE"
```

## 🔴 常见问题排查

### 问题 1: 小屏幕有横向滚动条
**原因**: 元素宽度 > 100%（如 `calc(100% + 32px)` 或 margin 超出）
**排查**:
```css
/* ❌ 错误 */
width: calc(100% + 32px);
margin-left: -16px;  /* 负 margin 也会溢出 */

/* ✅ 正确 */
width: 100%;
padding: 0 16px;
box-sizing: border-box;
```

### 问题 2: 分界线 .decoline 显示异常
**原因**: 高度、宽度或 margin 问题
**排查**:
```css
/* 确保分界线样式 */
.decoline {
    height: 3px;  /* 可见 */
    margin: 0;    /* 无外边距 */
    width: 100%;  /* 占满容器宽度 */
    background-color: #3C2CFF;  /* 有颜色 */
}
```

### 问题 3: 文字被遮挡或超出
**原因**: 字号或行高计算错误
**排查**:
```css
/* 小屏幕应该缩小字号 */
@media (max-width: 767px) {
    h1 { font-size: 44px; }  /* < 原始 93px */
    h3 { font-size: 32px; }  /* < 原始 62px */
    p  { font-size: 13px; }  /* < 原始 20px */
}
```

### 问题 4: 卡片不堆叠
**原因**: float 或 display 未重置
**排查**:
```css
/* 小屏幕必须去除 float */
.portfolio .desc {
    float: none;  /* ✅ 去除 */
    width: 100%;  /* ✅ 100% 宽 */
    margin-left: 0;  /* ✅ 无左边距 */
}
```

## 🧪 完整验证流程（约 5 分钟）

### 步骤 1: 启动本地服务器
```bash
npm start
# 或
http-server . -o
```
页面应在 http://localhost:8080 打开

### 步骤 2: 进入响应式设计模式
```
F12 → Ctrl+Shift+M
```

### 步骤 3: 依次测试各个宽度

| 宽度 | 设备 | 检查项 |
|------|------|---------|
| 375px | iPhone SE | 导航栏、首屏文字、3D 模型、无横向滚动 |
| 480px | Android | 作品卡片堆叠、分界线、边距 |
| 768px | iPad | 平滑过渡、布局变化 |
| 1024px | iPad Pro | 接近桌面样式 |
| 1200px+ | 桌面 | 原始布局不变 |

### 步骤 4: 检查控制台
```
F12 → Console
```
确保无红色错误信息

### 步骤 5: 截图对比（可选）
```bash
git show HEAD:css/index.css > /tmp/old.css
# 对比旧样式截图
```
